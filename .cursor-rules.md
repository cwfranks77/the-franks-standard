# Franks Standard isolation rules

**Project:** The Franks Standard & B&C Performance Audio  
**Canonical repo:** `C:\Users\ninja\Desktop\franks-restored`  
**GitHub:** `cwfranks77/the-franks-standard` (branch `master`)

These rules keep **two separate businesses** from bleeding into each other. Cursor and all agents must follow them on every task unless the owner explicitly names the other brand.

---

## Architecture locks (do not break)

1. **Framework:** This site runs on **Nuxt 3 (Vue.js)** only — not flat HTML, not a different framework.

2. **The Franks Standard** — marketplace facilitator (Amazon/eBay-style shell):
   - **Home:** `pages/index.vue` — high-density multi-vendor marketplace (large search bar, department dropdowns, item cards, seller badges).
   - **Scope:** `pages/` **except** `pages/bc-audio/**`, Franks ops, marketplace components, `layouts/default.vue`, Franks deploy workflows.

3. **B&C Performance Audio** — independent merchant storefront:
   - **Home:** `pages/bc-audio/index.vue` and everything under `pages/bc-audio/**`.
   - **Look:** Dark theme, neon-red accent `#d32f2f`, competition sound catalog grids.
   - **Namespace:** Stay inside the protected `pages/bc-audio/` folder and B&C-specific assets (`layouts/bc-audio.vue`, `components/Bc*.vue`, B&C composables/utils/plugins/middleware, Petra catalog, bcpoweraudio deploy).

4. **Never blend** identities, themes, copy, or UI between Franks and B&C in the same file.

5. **Sales tax:** Calculate from the customer’s **shipping destination ZIP**, not billing address (Louisiana marketplace facilitator / Act 22).

6. **Checkout reserves:** Implement **25% owner business income tax reserve** alongside wholesale transfers to fulfillment nodes where checkout logic applies.

7. **Budget safety:** No loops or commands that trigger unapproved API usage or on-demand server overcharges.

## Strict file discipline

Before any edit:

1. **Locate the correct file** — confirm path and whether it belongs to Franks, B&C, ops, or scripts.
2. **Read surrounding code** — match conventions; smallest correct diff only.
3. **Never edit the wrong file** — B&C tasks stay in `pages/bc-audio/**`; Franks marketplace tasks stay out of that folder.
4. **No drive-by refactors** — do not break unrelated files while fixing one issue.
5. **Verify after changes** — re-read edited sections; lint only touched files.
6. **Show what changed** before deploy or commit — owner reviews before live push unless explicitly told to push.
